## ВВЕДЕНИЕ В ПРОЕКТ

**TimQWees** - это полноценная система доставки еды, разработанная на фреймворке Django. Проект предназначен для курсовой работы и демонстрирует использование современных веб-технологий и лучших практик разработки.

### Что умеет система:

- **Управление ресторанами**: Создание, редактирование, удаление ресторанов
- **Управление продуктами**: Добавление блюд в меню ресторанов
- **Система заказов**: Клиенты могут оформлять заказы, отслеживать их статус
- **Корзина покупок**: Добавление товаров в корзину, изменение количества
- **REST API**: Полноценный API для интеграции с мобильными приложениями
- **Асинхронные задачи**: Отправка email уведомлений в фоне
- **OAuth2 авторизация**: Безопасная авторизация через токены
- **Социальная авторизация**: Вход через Google
- **Экспорт данных**: Выгрузка данных в Excel
- **История изменений**: Отслеживание всех изменений в базе данных
- **Профилирование**: Анализ производительности запросов

## УСТАНОВКА И НАСТРОЙКА ОКРУЖЕНИЯ

### Шаг 1: Клонирование проекта

```bash
git clone <repository-url>
cd kyrs
```

### Шаг 2: Создание виртуального окружения

**Виртуальное окружение** - это изолированная среда Python, где устанавливаются зависимости только для этого проекта. Это необходимо, чтобы разные проекты не конфликтовали друг с другом по версиям пакетов.

```bash
# Создание виртуального окружения
python3 -m venv venv

# Активация на macOS/Linux
source venv/bin/activate

# Активация на Windows
venv\Scripts\activate
```

После активации в терминале появится префикс `(venv)`, означающий, что вы работаете в виртуальном окружении.

### Шаг 3: Установка зависимостей

**requirements.txt** - файл со списком всех Python библиотек, необходимых для проекта.

```bash
pip install -r requirements.txt
```

Эта команда прочитает файл `requirements.txt` и установит все указанные пакеты с нужными версиями.

### Шаг 4: Настройка переменных окружения

**Переменные окружения** - это настройки, которые хранятся отдельно от кода. Это безопасно, так как секретные данные (пароли, ключи) не попадают в Git.

Скопируйте пример файла `.env.example` в `.env`:

```bash
cp .env.example .env
```

Откройте файл `.env` и отредактируйте его:

```env
# Django
DEBUG=True                                          # Режим отладки (True для разработки, False для продакшена)
SECRET_KEY=django-insecure-dev-key-change-in-production-12345  # Секретный ключ Django (смените в продакшене!)
ALLOWED_HOSTS=*                                     # Разрешенные хосты

# База данных (SQLite по умолчанию)
# Для PostgreSQL раскомментируйте строки ниже:
# DB_ENGINE=django.db.backends.postgresql
# DB_NAME=timqwees
# DB_USER=postgres
# DB_PASSWORD=postgres
# DB_HOST=db
# DB_PORT=5432

# Redis — брокер для Celery
CELERY_BROKER_URL=redis://localhost:6379/0

# Sentry — мониторинг ошибок
# Получите DSN на https://sentry.io и вставьте сюда
SENTRY_DSN=

# Email — Mailhog (тестовый SMTP)
EMAIL_HOST=localhost
EMAIL_PORT=1025
EMAIL_USE_TLS=False
EMAIL_HOST_USER=
EMAIL_HOST_PASSWORD=
DEFAULT_FROM_EMAIL=noreply@timqwees.com

# OAuth2 — создайте Application через /admin/oauth2_provider/application/
OAUTH2_CLIENT_ID=
OAUTH2_CLIENT_SECRET=
```

### Шаг 5: Применение миграций

**Миграции** - это способ Django применять изменения в структуре базы данных. Когда вы меняете модели, Django создает файлы миграций, которые затем применяются к базе данных.

```bash
# Создание миграций (если были изменения в моделях)
python manage.py makemigrations

# Применение миграций к базе данных
python manage.py migrate
```

### Шаг 6: Создание суперпользователя

Суперпользователь имеет полный доступ к админ-панели и всем функциям сайта.

```bash
python manage.py createsuperuser
```

Вам будет предложено ввести:
- Username (имя пользователя)
- Email address
- Password (пароль)
- Password confirmation (подтверждение пароля)

### Шаг 7: Запуск сервера разработки

```bash
python manage.py runserver 0.0.0.0:8080
```

Или используйте готовый скрипт:

```bash
./run.sh
```

Этот скрипт автоматически:
1. Создаст миграции
2. Применит миграции
3. Создаст суперпользователя (если не существует)
4. Создаст тестовые данные
5. Запустит сервер на порту 8080

### Доступ к приложению

- **Главный сайт**: http://localhost:8080
- **Админ-панель**: http://localhost:8080/admin
- **API**: http://localhost:8080/api/
- **Профилирование Silk**: http://localhost:8080/silk/

### 5. django-filter

**Что это:** Библиотека для фильтрации данных в Django REST Framework.

**Зачем нужен:**
- Фильтрация по полям модели
- Фильтрация по диапазонам (цена, дата)
- Комбинированные фильтры

**Где подключается:**
- В `requirements.txt`
- В `settings.py` в `INSTALLED_APPS` и `REST_FRAMEWORK`
- В `app/filters.py` - классы фильтров
- В `app/viewsets.py` - использование фильтров

```python
# app/filters.py
import django_filters

class ProductFilter(django_filters.FilterSet):
    min_price = django_filters.NumberFilter(field_name='price', lookup_expr='gte')
    max_price = django_filters.NumberFilter(field_name='price', lookup_expr='lte')
    
    class Meta:
        model = Product
        fields = ['restaurant', 'price']

# app/viewsets.py
class ProductViewSet(viewsets.ModelViewSet):
    filterset_class = ProductFilter
```

## МОДЕЛИ ДАННЫХ

### Что такое модели?

**Модели** - это Python классы, которые описывают структуру базы данных. Django ORM автоматически преобразует эти классы в таблицы в базе данных.

### Файл app/models.py

#### 1. Модель Restaurant (Ресторан)

```python
class Restaurant(models.Model):
    """Модель ресторана"""
    name = models.CharField(max_length=200, verbose_name="Название", db_index=True)
    address = models.CharField(max_length=300, verbose_name="Адрес")
    phone = models.CharField(max_length=20, verbose_name="Телефон")
    owner = models.ForeignKey(User, on_delete=models.CASCADE, verbose_name="Владелец")
    created_at = models.DateTimeField(auto_now_add=True, verbose_name="Дата создания")

    history = HistoricalRecords()

    class Meta:
        verbose_name = "Ресторан"
        verbose_name_plural = "Рестораны"
        ordering = ['name']
        indexes = [
            models.Index(fields=['name'], name='restaurant_name_idx'),
            models.Index(fields=['owner', 'created_at'], name='restaurant_owner_created_idx'),
        ]

    def __str__(self):
        return self.name
```

**Разбор полей:**

- **name** - название ресторана
  - `CharField` - текстовое поле
  - `max_length=200` - максимальная длина 200 символов
  - `verbose_name="Название"` - отображаемое имя в админке
  - `db_index=True` - создать индекс в БД для быстрого поиска

- **address** - адрес ресторана
  - `CharField` - текстовое поле
  - `max_length=300` - максимальная длина 300 символов

- **phone** - телефон
  - `CharField` - текстовое поле (не IntegerField, так как могут быть плюсы, скобки)

- **owner** - владелец ресторана
  - `ForeignKey` - внешний ключ на модель User
  - `on_delete=models.CASCADE` - при удалении пользователя удалить и его рестораны
  - Связь: один пользователь может иметь много ресторанов

- **created_at** - дата создания
  - `DateTimeField` - поле даты и времени
  - `auto_now_add=True` - автоматически устанавливается при создании

- **history** - история изменений
  - `HistoricalRecords()` - автоматически отслеживает все изменения

**Meta класс:**

- **verbose_name** - название в единственном числе
- **verbose_name_plural** - название во множественном числе
- **ordering** - сортировка по умолчанию
- **indexes** - индексы для ускорения запросов

**Метод __str__:**

- Возвращает строковое представление объекта
- Используется в админ-панели и при выводе

```python
class Product(models.Model):

    name = models.CharField(max_length=200, verbose_name="Название", db_index=True)
    description = models.TextField(verbose_name="Описание")
    price = models.DecimalField(max_digits=10, decimal_places=2, verbose_name="Цена")
    restaurant = models.ForeignKey(Restaurant, on_delete=models.CASCADE, verbose_name="Ресторан")
    created_at = models.DateTimeField(auto_now_add=True, verbose_name="Дата создания")

    history = HistoricalRecords()

    class Meta:
        verbose_name = "Продукт"
        verbose_name_plural = "Продукты"
        ordering = ['name']
        indexes = [
            models.Index(fields=['restaurant', 'name'], name='product_restaurant_name_idx'),
            models.Index(fields=['price'], name='product_price_idx'),
        ]

    def __str__(self):
        return f"{self.name} - {self.restaurant.name}"
```

**Разбор полей:**

  - `TextField` - текстовое поле без ограничения длины

  - `DecimalField` - поле для денежных сумм

```python
class Courier(models.Model):
    """Модель курьера"""
    user = models.OneToOneField(User, on_delete=models.CASCADE, verbose_name="Пользователь")
    phone = models.CharField(max_length=20, verbose_name="Телефон", db_index=True)
    is_active = models.BooleanField(default=True, verbose_name="Активен")

    class Meta:
        verbose_name = "Курьер"
        verbose_name_plural = "Курьеры"
        ordering = ['user__username']
        indexes = [
            models.Index(fields=['is_active'], name='courier_active_idx'),
        ]

    def __str__(self):
        return f"{self.user.username}"
```

**Разбор полей:**

- **user** - пользователь
  - `OneToOneField` - один к одному
  - Один пользователь может быть только одним курьером
- **phone** - телефон

**Сложная фильтрация с Q объектами:**

```python
def get_queryset(self) -> Any:
    """Фильтрация с использованием Q объектов"""
    queryset = super().get_queryset()

    min_price = self.request.query_params.get('min_price', None)
    max_price = self.request.query_params.get('max_price', None)

    q_objects = Q()

    if min_price:
        try:
            q_objects &= Q(price__gte=float(min_price))
        except (ValueError, TypeError):
            pass
    if max_price:
        try:
            q_objects &= Q(price__lte=float(max_price))
        except (ValueError, TypeError):
            pass

    if q_objects:
        queryset = queryset.filter(q_objects)

    return queryset.select_related('restaurant', 'restaurant__owner')
```

**Объяснение:**
- `Q` - объект для сложных запросов (AND, OR, NOT)
- `&=` - AND (И)
- `|=` - OR (ИЛИ)
- `~` - NOT (НЕ)
- `select_related` - оптимизация JOIN запросов

```python
@action(detail=False, methods=['get'])
def popular_products(self, request: Request, *args: Any, **kwargs: Any) -> Response:

        Q(orderitem__order__status__in=['pending', 'preparing', 'ready',
                                    'delivering', 'completed'])
        &
        ~Q(orderitem__order__status='cancelled')
```

```python
@action(detail=False, methods=['post'])
def bulk_discount(self, request: Request, *args: Any, **kwargs: Any) -> Response:
    """Массовая скидка с использованием F-выражений"""
    restaurant_id = request.data.get('restaurant_id')
    discount_percent = request.data.get('discount_percent', 10)

    multiplier = 1 - discount_percent / 100
    updated = Product.objects.filter(restaurant_id=restaurant_id).update(
        price=F('price') * multiplier
    )
    return Response({'message': f'Скидка {discount_percent}% применена к {updated} продуктам'})
```

**Объяснение:**

- Обновление происходит на уровне БД без загрузки в Python
- Очень быстро для большого количества записей

**Сложные запросы с Q:**

```python
def get_queryset(self) -> Any:
    """Фильтрация с использованием Q объектов (OR, AND, NOT)"""
    queryset = super().get_queryset()
    q_objects = Q()

    status_filter = self.request.query_params.get('status', None)
    if status_filter:
        q_objects &= Q(status=status_filter)

    date_from = self.request.query_params.get('date_from', None)
    date_to = self.request.query_params.get('date_to', None)
    if date_from:
        q_objects &= Q(created_at__gte=date_from)
    if date_to:
        q_objects &= Q(created_at__lte=date_to)

    # Сложный запрос: заказы с высоким приоритетом ИЛИ не завершенные
    high_priority = self.request.query_params.get('high_priority', None)
    if high_priority == 'true':
        q_objects |= Q(total_price__gte=1000) | ~Q(status__in=['completed', 'cancelled'])

    # Еще один сложный запрос: заказы, которые НЕ отменены И (готовятся ИЛИ доставляются)
    active_orders = self.request.query_params.get('active', None)
    if active_orders == 'true':
        q_objects &= ~Q(status='cancelled') & (Q(status='preparing') | Q(status='delivering'))

    if q_objects:
        queryset = queryset.filter(q_objects)

    return queryset.select_related('customer', 'restaurant', 'courier', 'courier__user').prefetch_related('items', 'items__product')
```

**Объяснение:**
- `prefetch_related` - оптимизация для ManyToMany и обратных ForeignKey
- `select_related` - для ForeignKey
- Комбинация Q объектов для сложной логики

**Action для изменения статуса:**

```python
@action(detail=True, methods=['post'])
def change_status(self, request: Request, pk: int = None, *args: Any, **kwargs: Any) -> Response:
    """Изменить статус заказа"""
    order = self.get_object()
    new_status = request.data.get('status', None)

    if new_status not in [choice[0] for choice in Order.STATUS_CHOICES]:
        return Response({'error': 'Недопустимый статус'}, status=status.HTTP_400_BAD_REQUEST)

    with transaction.atomic():
        order.status = new_status
        order.save()

    notify_order_status_change.delay(order.id, new_status)

    serializer = self.get_serializer(order)
    return Response(serializer.data)
```

**Объяснение:**
- `transaction.atomic()` - атомарная транзакция (все или ничего)

## ВАЛИДАЦИЯ ДАННЫХ

### Что такое валидация?

**Валидация** - проверка данных на корректность перед сохранением в базу данных.

### Файл app/serializers.py

**Сериализаторы** - преобразуют данные между Python объектами и JSON, а также валидируют данные.

#### 1. RestaurantSerializer

```python
class RestaurantSerializer(serializers.ModelSerializer):
    """Сериализатор для ресторана"""
    owner_username = serializers.CharField(source='owner.username', read_only=True)

    class Meta:
        model = Restaurant
        fields = ['id', 'name', 'address', 'phone', 'owner', 'owner_username', 'created_at']
        read_only_fields = ['created_at']

    def validate_name(self, value):
        """Валидация: название ресторана должно быть уникальным"""
        if self.instance and self.instance.name == value:
            return value
        if Restaurant.objects.filter(name=value).exists():
            raise serializers.ValidationError("Ресторан с таким названием уже существует")
        return value

    def validate_phone(self, value):
        """Валидация: телефон должен содержать только цифры"""
        if not value.replace('+', '').replace('-', '').replace(' ', '').isdigit():
            raise serializers.ValidationError("Телефон должен содержать только цифры и допустимые символы")
        return value
```

**Объяснение:**
- `ModelSerializer` - базовый класс для сериализаторов моделей
- `owner_username` - дополнительное поле только для чтения
- `validate_*` - методы валидации конкретных полей
- `ValidationError` - исключение при ошибке валидации

#### 2. ProductSerializer

```python
class ProductSerializer(serializers.ModelSerializer):
    """Сериализатор для продукта"""
    restaurant_name = serializers.CharField(source='restaurant.name', read_only=True)
    is_in_cart = serializers.SerializerMethodField()

    class Meta:
        model = Product
        fields = ['id', 'name', 'description', 'price', 'restaurant', 'restaurant_name', 'created_at', 'is_in_cart']
        read_only_fields = ['created_at']

    def get_is_in_cart(self, obj: Product) -> bool:
        """Проверяет, находится ли продукт в корзине текущего пользователя"""
        cart_product_ids = self.context.get('cart_product_ids', [])
        return obj.id in cart_product_ids

    def validate_price(self, value):
        """Валидация: цена должна быть положительной"""
        if value <= 0:
            raise serializers.ValidationError("Цена должна быть больше нуля")
        if value > 100000:
            raise serializers.ValidationError("Цена не может превышать 100000")
        return value

    def validate_name(self, value):
        """Валидация: название продукта должно быть уникальным в рамках ресторана"""
        restaurant = self.initial_data.get('restaurant')
        if restaurant:
            if self.instance and self.instance.restaurant_id == restaurant and self.instance.name == value:
                return value
            if Product.objects.filter(restaurant_id=restaurant, name=value).exists():
                raise serializers.ValidationError("Продукт с таким названием уже существует в этом ресторане")
        return value
```

**Объяснение:**
- `SerializerMethodField` - вычисляемое поле
- `get_*` - метод для вычисления значения
- `self.context` - контекст сериализатора (передается из ViewSet)
- `self.initial_data` - входные данные

```python
    """Сериализатор для заказа"""
    customer_username = serializers.CharField(source='customer.username', read_only=True)
    restaurant_name = serializers.CharField(source='restaurant.name', read_only=True)
    courier_username = serializers.CharField(source='courier.user.username', read_only=True, allow_null=True)
    items = OrderItemSerializer(many=True, read_only=True)
    status_display = serializers.CharField(source='get_status_display', read_only=True)

    class Meta:
        model = Order
        fields = [
            'id', 'customer', 'customer_username', 'restaurant', 'restaurant_name',
            'courier', 'courier_username', 'status', 'status_display', 'address',
            'total_price', 'created_at', 'updated_at', 'items'
        ]
        read_only_fields = ['total_price', 'created_at', 'updated_at']

    def validate_address(self, value):
        """Валидация: адрес не должен быть пустым"""
        if not value or len(value.strip()) < 10:
            raise serializers.ValidationError("Адрес должен содержать минимум 10 символов")
        return value

    def validate(self, data):
        """Валидация: проверка общей логики заказа"""
        if 'status' in data:
            status = data['status']
```

## АСИНХРОННЫЕ ЗАДАЧИ (CELERY)

### Что такое Celery?

**Celery** - библиотека для асинхронного выполнения задач. Позволяет выполнять долгие операции в фоне, не блокируя основной поток.

### Зачем нужен Celery?

- Отправка email (может занимать время)
- Обработка изображений
- Периодические задачи (очистка старых данных)
- Генерация отчетов

### Архитектура Celery:

```
Django App ----> Celery ----> Redis (брокер)
                              |
                              v
                         Celery Worker
                              |
                              v
                         Выполнение задачи
```

### Файл timqwees/celery.py

```python
"""Celery конфигурация для проекта timqwees"""
import os
from celery import Celery

# Указываем настройки Django
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'timqwees.settings')

app = Celery('timqwees')

# Загружаем настройки из settings.py (все переменные с префиксом CELERY_)
app.config_from_object('django.conf:settings', namespace='CELERY')

# Автоматическое обнаружение tasks.py в каждом приложении
app.autodiscover_tasks()
```

**Объяснение:**
- `Celery('timqwees')` - создание экземпляра Celery
- `config_from_object` - загрузка настроек из Django settings
- `autodiscover_tasks` - автоматическое обнаружение задач

### Файл app/tasks.py

```python
"""Celery задачи для приложения"""
from celery import shared_task
from django.core.mail import send_mail
from django.utils import timezone
from datetime import timedelta

@shared_task
def send_order_confirmation_email(order_id: int, email: str, username: str) -> str:
    """Асинхронная отправка письма подтверждения заказа"""
    send_mail(
        subject=f'Заказ #{order_id} оформлен!',
        message=f'Здравствуйте, {username}! Ваш заказ #{order_id} успешно оформлен.',
        from_email='noreply@timqwees.com',
        recipient_list=[email],
        fail_silently=False,
    )
    return f'Письмо отправлено для заказа #{order_id}'
```

**Объяснение:**
- `@shared_task` - декоратор для создания задачи
- Задача может быть вызвана из любого приложения
- `send_mail` - отправка email через Django

```python
@shared_task
def cleanup_old_orders(days: int = 90) -> str:
    """Периодическая задача: удаление старых отмененных заказов"""
    from app.models import Order
    cutoff_date = timezone.now() - timedelta(days=days)
    old_orders = Order.objects.filter(
        status='cancelled',
        created_at__lt=cutoff_date
    )
    count = old_orders.count()
    old_orders.delete()
    return f'Удалено {count} старых отмененных заказов'
```

**Объяснение:**
- Периодическая задача для очистки старых данных
- `timedelta` - разница во времени

```python
@shared_task
def notify_order_status_change(order_id: int, new_status: str) -> str:
    """Уведомление об изменении статуса заказа"""
    from app.models import Order
    try:
        order = Order.objects.select_related('customer').get(id=order_id)
        status_display = dict(Order.STATUS_CHOICES).get(new_status, new_status)
        send_mail(
            subject=f'Статус заказа #{order_id} изменен',
            message=f'Статус вашего заказа #{order_id} изменен на: {status_display}',
            from_email='noreply@timqwees.com',
            recipient_list=[order.customer.email] if order.customer.email else [],
            fail_silently=True,
        )
        return f'Уведомление отправлено для заказа #{order_id}'
    except Order.DoesNotExist:
        return f'Заказ #{order_id} не найден'
```

## ЭКСПОРТ ДАННЫХ В EXCEL

### Что такое экспорт?

**Экспорт** - выгрузка данных из базы данных в файл (Excel, CSV).

### Библиотека django-import-export

Позволяет легко экспортировать и импортировать данные.

### Ресурс для экспорта (app/admin.py)

```python
from import_export import resources
from import_export.fields import Field

class ProductResource(resources.ModelResource):
    """Ресурс для экспорта продуктов"""
    restaurant_name = Field(attribute='restaurant__name', column_name='Ресторан')
    formatted_price = Field(column_name='Цена (форматированная)')

    class Meta:
        model = Product
        fields = ('id', 'name', 'description', 'price', 'restaurant_name')
        export_order = ('id', 'name', 'description', 'restaurant_name',
                        'price', 'formatted_price', 'created_at')

    def dehydrate_formatted_price(self, product):
        """Форматирование цены"""
        return f"{product.price} ₽"

    def get_export_queryset(self):
        """Экспорт только продуктов с ценой выше 100"""
        return Product.objects.filter(price__gte=100)
```

**Объяснение:**
- `Field` - дополнительное поле
- `attribute` - путь к атрибуту (можно использовать `__` для ForeignKey)
- `column_name` - название колонки в Excel
- `dehydrate_*` - метод для форматирования значения
- `get_export_queryset` - фильтрация данных для экспорта
- `export_order` - порядок колонок

```python
from app.models import Restaurant
```

### Оптимизация запросов

#### Проблема: N+1 запросов

```python
# Медленно (N+1 запросов)
orders = Order.objects.all()
for order in orders:
    print(order.restaurant.name)  # Отдельный запрос для каждого заказа
```

#### Решение: select_related

```python
# Быстро (1 запрос)
orders = Order.objects.select_related('restaurant').all()
for order in orders:
    print(order.restaurant.name)
```

#### Решение: prefetch_related

```python
# Быстро для ManyToMany
orders = Order.objects.prefetch_related('items').all()
for order in orders:
    for item in order.items.all():
        print(item.product.name)
```

```python
# app/management/commands/cleanup_old_orders.py
from django.core.management.base import BaseCommand
from django.utils import timezone
from datetime import timedelta
from app.models import Order

class Command(BaseCommand):
    help = 'Удаляет старые отмененные заказы (старше 90 дней)'

    def add_arguments(self, parser):
        parser.add_argument(
            '--days',
            type=int,
            default=90,
            help='Количество дней для удаления заказов (по умолчанию 90)',
        )

    def handle(self, *args, **options):
        days = options['days']
        cutoff_date = timezone.now() - timedelta(days=days)

        old_orders = Order.objects.filter(
            status='cancelled',
            created_at__lt=cutoff_date
        )

        count = old_orders.count()
        old_orders.delete()

        self.stdout.write(
            self.style.SUCCESS(f'Удалено {count} отмененных заказов старше {days} дней')
        )
```

```python
        """Подготовка данных"""
        self.client = APIClient()
        self.user = User.objects.create_user(username='testuser', password='testpass')
        self.restaurant = Restaurant.objects.create(
            name='Тестовый ресторан',
            address='ул. Тестовая, 1',
            phone='+79991234567',
            owner=self.user
        )
        self.product = Product.objects.create(
            name='Пицца',
            description='Вкусная пицца',
            price=500,
            restaurant=self.restaurant
        )

    def test_get_products_list(self):
        """Тест получения списка продуктов"""
        response = self.client.get('/api/products/')
        self.assertEqual(response.status_code, status.HTTP_200_OK)
        self.assertEqual(len(response.data), 1)

    def test_create_product(self):
        """Тест создания продукта"""
        self.client.force_authenticate(user=self.user)
        data = {
            'name': 'Бургер',
            'description': 'Вкусный бургер',
            'price': 350,
            'restaurant': self.restaurant.id
        }
        response = self.client.post('/api/products/', data)
        self.assertEqual(response.status_code, status.HTTP_201_CREATED)
        self.assertEqual(Product.objects.count(), 2)
```

**Объяснение:**
- `TestCase` - базовый класс для тестов
- `setUp()` - подготовка данных перед каждым тестом
- `assertEqual()` - проверка равенства
- `APIClient` - клиент для тестирования API
- `force_authenticate()` - авторизация клиента
