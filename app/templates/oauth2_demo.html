OAuth2 — Протокол авторизации

Полный цикл OAuth2: создание приложения, авторизация, получение токена, использование API

Как работает OAuth2

1

Регистрация приложения

Создайте OAuth2 Application через админ-панель. Получите client_id и client_secret.

2

Запрос авторизации

Клиент перенаправляется на /api/o/authorize/ с параметрами response_type=code, client_id, redirect_uri, scope.

{{ base_url }}api/o/authorize/?response_type=code&client_id=YOUR_CLIENT_ID&redirect_uri={{ base_url }}oauth2/callback/&scope=read+write
3

Получение authorization code

После одобрения пользователя, сервер редиректит на redirect_uri с параметром code.

4

Обмен code на access_token

POST-запрос на /api/o/token/ с grant_type=authorization_code, code, redirect_uri, client_id, client_secret.

curl -X POST {{ base_url }}api/o/token/ \ -d "grant_type=authorization_code" \ -d "code=AUTH_CODE" \ -d "redirect_uri={{ base_url }}oauth2/callback/" \ -d "client_id=YOUR_CLIENT_ID" \ -d "client_secret=YOUR_CLIENT_SECRET"
5

Использование токена для API

Добавьте заголовок Authorization: Bearer ACCESS_TOKEN к запросам API.

curl -H "Authorization: Bearer YOUR_TOKEN" {{ base_url }}api/restaurants/
